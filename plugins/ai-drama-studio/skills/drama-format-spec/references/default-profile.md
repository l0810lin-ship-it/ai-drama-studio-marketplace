# 团队默认格式建议

这些值是缺少客户、项目和平台规格时的起始建议，不是全局硬规则。

| 项目 | 默认建议 |
|---|---|
| 集纲 | 每集一段连续叙事，结尾单列视觉钩子 |
| 剧本动作 | 使用一致动作行标记，写可见、可生成的动作 |
| 台词 | 移动端可读、适合口型；超长句标记复核，不自动删改 |
| 场景 | 优先控制单集场景切换，但阈值由时长与客户样例决定 |
| 钩子 | 每集末明确最后一句或最后一帧 |
| 修订交付 | 默认不保留内部批注、修订痕迹和内部色标 |
| 文件 | 默认 DOCX；文件名规则以客户系统兼容性为准 |

历史材料中出现过集纲 254–363 字、剧本 600–800 字、单句台词不超过 25 字、单集不超过 5 场等阈值。除非项目证据确认，否则只作为待验证候选，不用于阻断交付。
