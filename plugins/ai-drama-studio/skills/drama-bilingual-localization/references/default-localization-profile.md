# 本地化起始建议

本文件只提供无项目证据时的起始建议，不是 Project Bible，也不能覆盖客户样例、锁定术语或批准决定。

## 默认处理范围

- 优先自然、简洁、可表演的目标语言，不使用机械直译。
- 对白与所有剧情可读的屏幕文字默认都进入翻译范围。动作行、场景标题和制作说明是否翻译，由活动配置决定。
- 屏幕文字包括短信、聊天、联系人/来电显示、通知、社媒、邮件、信件、合同、新闻标题、招牌和剧情相关 UI。纯装饰且不可读的界面可以不译，但要在台账标记 `decorative`。
- “只翻台词、动作行保留源语言”只是常见双语格式之一；它不等于可以漏掉藏在动作行里的手机屏幕文字。

## 语言与版面

- 目标语言长度应适配字幕、口型、聊天气泡和道具画面，但具体字符/行数阈值由交付规格决定。
- 一句先保住信息功能、情绪动作和人物关系，再处理自然度；不能为了短而删掉证据、威胁条件、关系称谓或剧情钩子。
- 称谓、类型术语和文化等价物需要查证，不凭语感宣布“地道”。同一角色对不同对象的称谓要体现权力与亲疏变化。
- 短信与聊天允许符合目标市场的缩写、断句和表情，但必须匹配角色年龄、职业、阶层、时代与情境；正式通知、合同、新闻和系统 UI 使用各自文体。

## 世界观与市场

- 先读项目世界观和时代，再决定术语、称谓、语域、媒介和技术。世界观不仅约束名词，也约束“谁能说什么、通过什么渠道说”。
- 受众年龄、地域、平台和评级必须来自项目或市场证据。
- 文化替换要记录 `source_evidence` 与影响；无法确认时保留原意并标注待审，不擅自把某一地区习惯写成全球默认。

## 默认复核

- 完整性：每条对白和剧情可读文字都有状态。
- 一致性：术语、姓名、称谓、标点、大小写与角色声音一致。
- 合理性：技术、年代、通信渠道和世界规则不冲突。
- 可演/可见性：台词可说，字幕可读，短信与 UI 能装入既定画面。
- 可追溯性：译文能回指源文行号、场次或镜头；推断与批准项分开记录。
